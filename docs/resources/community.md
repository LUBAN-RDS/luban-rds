## 3. 贡献指南

### 3.1 如何贡献

- **代码贡献**：提交 Pull Request 到 GitHub 仓库
- **文档贡献**：改进和完善项目文档
- **问题反馈**：在 Issues 页面报告 bug 或提出功能建议
- **测试贡献**：编写和运行测试用例
- **社区支持**：回答社区问题，参与讨论

### 3.2 开发流程

1. **Fork 仓库**：在 GitHub 上 Fork 代码仓库
2. **克隆代码**：克隆到本地开发环境
3. **创建分支**：创建功能分支或修复分支
4. **开发代码**：实现功能或修复 bug
5. **运行测试**：确保测试通过
6. **提交代码**：提交代码并推送到远程仓库
7. **创建 PR**：在 GitHub 上创建 Pull Request
8. **代码审查**：等待维护者审查代码
9. **合并代码**：代码通过审查后合并到主分支

### 3.3 编码规范

- **Java 版本**：使用 Java 8+ 语法
- **代码风格**：遵循 Google Java 风格指南
- **命名规范**：使用有意义的变量和方法名
- **文档注释**：为公共类和方法添加 Javadoc
- **测试覆盖**：为新功能添加单元测试

### 3.4 提交规范

- **提交消息**：使用清晰、简洁的提交消息
- **消息格式**：
  ```
  <type>: <description>
  
  <body>
  
  <footer>
  ```
- **类型**：feat（新功能）、fix（修复）、docs（文档）、style（代码风格）、refactor（重构）、test（测试）、chore（构建/依赖）
- **描述**：简短描述更改的内容
- **正文**：详细描述更改的原因和影响
- **脚注**：引用相关的 Issues 或 PR
